# دعواتي (مناسبتي) — Flutter Project

## حالة البناء (بعد الإصلاح)

المشروع كان يفشل في GitHub Actions برسالة:

```
Could not find or load main class org.gradle.wrapper.GradleWrapperMain
```

السبب: `android/gradle/wrapper/gradle-wrapper.jar` (ملف ثنائي حقيقي) لم يكن
موجودًا إطلاقًا في الأرشيف، رغم وجود `gradlew` / `gradlew.bat` اللذين يحتاجانه
ليعملا. تم إصلاح ذلك عبر:

- تحديث `android/gradle/wrapper/gradle-wrapper.properties` إلى Gradle
  **8.10.2**، و`android/settings.gradle.kts` إلى **AGP 8.7.3 + Kotlin 2.1.0**
  (مجموعة متوافقة ومُختبرة مع Flutter 3.35+).
- تحديث Java/Kotlin toolchain في `android/app/build.gradle.kts` إلى 17 (مطابق
  لمتطلبات AGP 8.7.x) وتفعيل `coreLibraryDesugaring` (يحتاجه
  `flutter_local_notifications`/`timezone` على `minSdk` أقل من 26).
- إعادة كتابة `.github/workflows/build_apk.yml` بالكامل بحيث:
  1. يُثبّت JDK 17 ثم Flutter (قناة stable).
  2. يُشغّل `flutter create --platforms=android .` (آمن تمامًا، لا يحذف
     `lib/`/`pubspec.yaml`، فقط يكمل أي ملف ناقص).
  3. **الخطوة الحاسمة**: يُثبّت Gradle حقيقي عبر
     `gradle/actions/setup-gradle@v4` ثم يُشغّل
     `gradle wrapper --gradle-version 8.10.2 --distribution-type all` داخل
     `android/` — هذا يُعيد كتابة `gradle-wrapper.jar` من الصفر بشكل مضمون،
     بدل الاعتماد على وجوده مسبقًا في المستودع.
  4. يتحقق أن الملف الناتج غير فارغ (`test -s gradle-wrapper.jar`) وأن
     `./gradlew --version` يعمل، **قبل** المتابعة إلى `flutter pub get` والبناء.

> ⚠️ **قيد بيئة الإصلاح**: هذا الإصلاح جرى داخل بيئة Sandbox بلا اتصال إنترنت
> فعلي (لا تحميل حزم pub، لا تحميل توزيعة Gradle، ولا حتى `javac` متاح لبناء
> jar يدويًا). لذلك لم يكن ممكنًا تشغيل `flutter pub get` أو
> `flutter build apk` هنا فعليًا للتحقق النهائي. الإصلاح أعلاه يعتمد على أن
> **GitHub Actions نفسه** لديه اتصال إنترنت كامل، فيقوم هو بتحميل Gradle
> الحقيقي وتوليد الـ wrapper وبناء الـ APK. إن أردت البناء محليًا بدل CI، راجع
> قسم "البناء محليًا" أدناه.

## البناء محليًا (إن كان لديك Flutter SDK + إنترنت)

```bash
flutter pub get
cd android
# يولّد gradle-wrapper.jar صحيحًا إن لم يكن موجودًا (يتطلب تثبيت Gradle مسبقًا،
# أو استخدم Android Studio: افتح المجلد وسيعرض "Sync" الذي يقوم بنفس الشيء)
gradle wrapper --gradle-version 8.10.2 --distribution-type all
cd ..
flutter build apk --release
```

إن لم يكن لديك `gradle` مثبتًا محليًا، الأبسط هو فتح مجلد `android/` بـ Android
Studio وسيصلح الـ wrapper تلقائيًا عند أول "Gradle Sync".

هذا مشروع Flutter فعلي (وليس مجرد مخطط) يطبّق أغلب ما ورد بمخطط البناء، بالاعتماد
على الافتراضات التالية (لأنك طلبت المتابعة قبل الإجابة على أسئلة المخطط):

1. **تسجيل الدخول محلي بالكامل (Mock)** — لا يوجد Firebase/Supabase فعلي؛
   `AuthRepository` بسيط يخزّن اسم المستخدم بـ `SharedPreferences`. الواجهة
   والتدفق (`/login → /onboarding → /home`, وزر "تسجيل الخروج") جاهزة تمامًا،
   لذا ربط باكند حقيقي لاحقًا يعني فقط إعادة كتابة محتوى هذا الملف الواحد
   (`lib/data/repositories/auth_repository.dart`) دون لمس أي شاشة أخرى.
2. **الكردية = السورانية بالأحرف العربية (RTL)** — إن كنت تقصد الكرمانجية
   باللاتينية (LTR)، عدّل `isRtl` في `lib/app.dart` وترجم ملف `app_ku.arb`.
3. **"عدد أفراد العائلة" خاصية للشخص نفسه**، تُستخدم كقيمة افتراضية لحقل
   "عدد المرافقين" عند دعوته لمناسبة (قابل للتعديل بعدها بمعزل عن الأول).
4. **تصدير PDF/Excel**: الحزم مضافة بـ `pubspec.yaml` (`pdf`, `printing`,
   `excel`) والأزرار موجودة بشاشتي الإعدادات والتقارير، لكن التنفيذ الفعلي
   متروك كـ TODO بسيط (`ScaffoldMessenger` مؤقت) لتبقى المراجعة الأولى
   قابلة للتشغيل مباشرة دون تعقيد إضافي — راجع القسم أدناه لإكمالها.

## لماذا لا يوجد drift/freezed/build_runner؟

تعمّدت استخدام **sqflite بجداول SQL مكتوبة يدويًا** ونماذج Dart عادية
(`fromMap`/`toMap`/`copyWith` مكتوبة يدويًا) بدل drift/freezed/riverpod_generator،
لأن بيئة التطوير التي كتبت بها هذا الكود **لا تملك Flutter SDK ولا اتصال
إنترنت**، وبالتالي لا يمكنني تشغيل `build_runner` للتحقق من أن الأكواد المولّدة
صحيحة. هذا الاختيار يعني أن المشروع **يعمل فور `flutter pub get` بدون أي خطوة
codegen إضافية**. إن رغبت بالتحويل لـ drift/freezed لاحقًا (لفوائد أمان الأنواع
وتوليد أقل كود يدوي)، فالبنية (Repository per domain) تسهّل ذلك دون تغيير الشاشات.

## خطوات التشغيل

هذا المجلد يحتوي `lib/` و`pubspec.yaml` و`l10n.yaml` فقط (لا مجلدات
`android/`/`ios native` لأن إنشاءها يتطلب Flutter SDK الذي لم يتوفر هنا). لتشغيله:

```bash
# 1) أنشئ هيكل مشروع Flutter فارغ بجانب هذا الكود (أو في نفس المجلد)
flutter create dawakti_shell
cd dawakti_shell

# 2) انسخ محتويات هذا الأرشيف فوقه (lib/, pubspec.yaml, l10n.yaml, analysis_options.yaml)
#    ثم وافق على استبدال lib/main.dart وpubspec.yaml

# 3) نزّل الحزم
flutter pub get

# 4) شغّل
flutter run
```

> ملاحظة: عند أول `flutter pub get` (بفضل `generate: true` بـ `pubspec.yaml`)
> سيولّد Flutter تلقائيًا صنف `AppLocalizations` من ملفات `lib/l10n/*.arb`.
> الشاشات الحالية لا تستخدم `AppLocalizations.of(context)` بعد (النصوص عربية
> مباشرة بالكود) — استبدالها تدريجيًا بمناداة `AppLocalizations` هو خطوة ميكانيكية
> لاحقة لتفعيل تعدد اللغات فعليًا في كل الشاشات وليس فقط بالبنية التحتية.

## ما هو مكتمل فعليًا (شاشات حقيقية بمنطق كامل)

- تسجيل دخول محلي + Splash + Onboarding (4 شاشات متحركة).
- الرئيسية: إحصائيات + Donut chart للفئات + **رسم بياني خطي "المناسبات هذا
  الشهر" مع قائمة اختيار الشهر** + قائمة مناسبات قادمة + إجراءات سريعة.
- الفئات: عرض/إضافة/تعديل/حذف + بحث + **قائمة "..." لكل بطاقة** (تعديل/حذف) +
  عدّاد أشخاص لكل فئة.
- الأشخاص: قائمة + بحث + فلاتر (الكل/المفضلة/**الأقسام**) + **فلتر متقدم**
  (بدون فئة/إعادة تعيين) + **قائمة "..." لكل صف** (تعديل/حذف) + نموذج
  إضافة/تعديل كامل (بما فيها ستيبر "عدد أفراد العائلة") + صفحة تفاصيل مع سجل
  المناسبات.
- المناسبات: قائمة مفلترة (الكل/مقبلة/جارية/منتهية) ببطاقات **صورة غلاف كبيرة**
  (مطابقة للمخطط) + قائمة "..." لكل بطاقة + نموذج إنشاء/تعديل كامل.
- اختيار المدعوين: تحديد/فلترة/بحث + ستيبر مرافقين لكل شخص + **قائمة حالة
  الدعوة المنسدلة (مدعو/لم يتم التواصل/معتذر/قيد الانتظار) أثناء التحديد
  مباشرة**، مطابقةً لتدفق المخطط.
- تفاصيل المناسبة: 3 تبويبات (نظرة عامة ببطاقة ملخص + المدعوين بأربع حالات
  RSVP وستيبر تعديل حي + المهام الخاصة بالمناسبة) + **أزرار تصدير Excel/PDF
  فعلية لكل مناسبة**.
- المهام: حلقة تقدّم + فلاتر + سحب لإعادة الترتيب + إضافة/تعديل/حذف +
  **صورة مصغّرة لكل مهمة (اختيار من المعرض + عرضها بالقائمة)**.
- التقارير: ملخص شامل على مستوى التطبيق كامل + **أزرار تصدير Excel/PDF فعلية**.
- الإعدادات: وضع ليلي، 3 لغات (تبديل فعلي للـ Locale)، لون تمييز،
  **استيراد/تصدير Excel فعلي، تصدير PDF فعلي**، رابط الأمان (**قفل PIN فعلي
  يُطبَّق فعليًا عند بدء التطبيق عبر شاشة `/lock` + بصمة/فيس آيدي اختيارية عبر
  local_auth**)، رابط "عن التطبيق"، تسجيل خروج فعلي.
- **إشعارات محلية مجدولة** لكل مناسبة (تذكير قبل يوم وقبل 3 ساعات)، تُنشأ/تُحدَّث/
  تُلغى تلقائيًا مع إنشاء/تعديل/حذف/نسخ المناسبة.

## ملاحظة حول إصدار Flutter SDK

كُتب `lib/core/theme/app_theme.dart` باستخدام `CardThemeData` (الاسم الحالي في
إصدارات Flutter الحديثة). إن كان إصدار Flutter لديك أقدم ويطلب `CardTheme` بدلاً
منه، فهذا مجرد اختلاف تسمية بين إصدارين واستبدال الاسم في ذلك الملف يكفي لحل
الخطأ — لم أتمكن من التحقق محليًا لأن بيئة الكتابة لا تملك Flutter SDK مثبتًا.
نفس التنبيه ينطبق على استدعاءات حزم `excel`/`pdf`/`printing` المُعلَّمة بتعليقات
`NOTE` داخل `lib/data/services/`.

## هيكلة المجلدات



راجع تعليق أعلى `lib/core/router/app_router.dart` و`lib/providers/providers.dart`
لفهم كيف تتصل الشاشات بالمستودعات (Repositories) عبر Riverpod. كل مستودع
(`*_repository.dart`) هو نقطة الاتصال الوحيدة بقاعدة sqflite — أي تبديل لاحق
لقاعدة بيانات أخرى (drift/Isar) أو لباكند سحابي يمر من هناك فقط.
